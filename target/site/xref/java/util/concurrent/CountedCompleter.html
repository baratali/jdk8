36  package java.util.concurrent;
37  
38  /**
39   * A {@link ForkJoinTask} with a completion action performed when
40   * triggered and there are no remaining pending actions.
41   * CountedCompleters are in general more robust in the
42   * presence of subtask stalls and blockage than are other forms of
43   * ForkJoinTasks, but are less intuitive to program.  Uses of
44   * CountedCompleter are similar to those of other completion based
45   * components (such as {@link java.nio.channels.CompletionHandler})
46   * except that multiple <em>pending</em> completions may be necessary
47   * to trigger the completion action {@link #onCompletion(CountedCompleter)},
48   * not just one.
49   * Unless initialized otherwise, the {@linkplain #getPendingCount pending
50   * count} starts at zero, but may be (atomically) changed using
51   * methods {@link #setPendingCount}, {@link #addToPendingCount}, and
52   * {@link #compareAndSetPendingCount}. Upon invocation of {@link
53   * #tryComplete}, if the pending action count is nonzero, it is
54   * decremented; otherwise, the completion action is performed, and if
55   * this completer itself has a completer, the process is continued
56   * with its completer.  As is the case with related synchronization
57   * components such as {@link java.util.concurrent.Phaser Phaser} and
58   * {@link java.util.concurrent.Semaphore Semaphore}, these methods
59   * affect only internal counts; they do not establish any further
60   * internal bookkeeping. In particular, the identities of pending
61   * tasks are not maintained. As illustrated below, you can create
62   * subclasses that do record some or all pending tasks or their
63   * results when needed.  As illustrated below, utility methods
64   * supporting customization of completion traversals are also
65   * provided. However, because CountedCompleters provide only basic
66   * synchronization mechanisms, it may be useful to create further
67   * abstract subclasses that maintain linkages, fields, and additional
68   * support methods appropriate for a set of related usages.
69   *
70   * <p>A concrete CountedCompleter class must define method {@link
71   * #compute}, that should in most cases (as illustrated below), invoke
72   * {@code tryComplete()} once before returning. The class may also
73   * optionally override method {@link #onCompletion(CountedCompleter)}
74   * to perform an action upon normal completion, and method
75   * {@link #onExceptionalCompletion(Throwable, CountedCompleter)} to
76   * perform an action upon any exception.
77   *
78   * <p>CountedCompleters most often do not bear results, in which case
79   * they are normally declared as {@code CountedCompleter<Void>}, and
80   * will always return {@code null} as a result value.  In other cases,
81   * you should override method {@link #getRawResult} to provide a
82   * result from {@code join(), invoke()}, and related methods.  In
83   * general, this method should return the value of a field (or a
84   * function of one or more fields) of the CountedCompleter object that
85   * holds the result upon completion. Method {@link #setRawResult} by
86   * default plays no role in CountedCompleters.  It is possible, but
87   * rarely applicable, to override this method to maintain other
88   * objects or fields holding result data.
89   *
90   * <p>A CountedCompleter that does not itself have a completer (i.e.,
91   * one for which {@link #getCompleter} returns {@code null}) can be
92   * used as a regular ForkJoinTask with this added functionality.
93   * However, any completer that in turn has another completer serves
94   * only as an internal helper for other computations, so its own task
95   * status (as reported in methods such as {@link ForkJoinTask#isDone})
96   * is arbitrary; this status changes only upon explicit invocations of
97   * {@link #complete}, {@link ForkJoinTask#cancel},
98   * {@link ForkJoinTask#completeExceptionally(Throwable)} or upon
99   * exceptional completion of method {@code compute}. Upon any
100  * exceptional completion, the exception may be relayed to a task's
101  * completer (and its completer, and so on), if one exists and it has
102  * not otherwise already completed. Similarly, cancelling an internal
103  * CountedCompleter has only a local effect on that completer, so is
104  * not often useful.
105  *
106  * <p><b>Sample Usages.</b>
107  *
108  * <p><b>Parallel recursive decomposition.</b> CountedCompleters may
109  * be arranged in trees similar to those often used with {@link
110  * RecursiveAction}s, although the constructions involved in setting
111  * them up typically vary. Here, the completer of each task is its
112  * parent in the computation tree. Even though they entail a bit more
113  * bookkeeping, CountedCompleters may be better choices when applying
114  * a possibly time-consuming operation (that cannot be further
115  * subdivided) to each element of an array or collection; especially
116  * when the operation takes a significantly different amount of time
117  * to complete for some elements than others, either because of
118  * intrinsic variation (for example I/O) or auxiliary effects such as
119  * garbage collection.  Because CountedCompleters provide their own
120  * continuations, other threads need not block waiting to perform
121  * them.
122  *
123  * <p>For example, here is an initial version of a class that uses
124  * divide-by-two recursive decomposition to divide work into single
125  * pieces (leaf tasks). Even when work is split into individual calls,
126  * tree-based techniques are usually preferable to directly forking
127  * leaf tasks, because they reduce inter-thread communication and
128  * improve load balancing. In the recursive case, the second of each
129  * pair of subtasks to finish triggers completion of its parent
130  * (because no result combination is performed, the default no-op
131  * implementation of method {@code onCompletion} is not overridden).
132  * A static utility method sets up the base task and invokes it
133  * (here, implicitly using the {@link ForkJoinPool#commonPool()}).
134  *
135  * <pre> {@code
136  * class MyOperation<E> { void apply(E e) { ... }  }
137  *
138  * class ForEach<E> extends CountedCompleter<Void> {
139  *
140  *   public static <E> void forEach(E[] array, MyOperation<E> op) {
141  *     new ForEach<E>(null, array, op, 0, array.length).invoke();
142  *   }
143  *
144  *   final E[] array; final MyOperation<E> op; final int lo, hi;
145  *   ForEach(CountedCompleter<?> p, E[] array, MyOperation<E> op, int lo, int hi) {
146  *     super(p);
147  *     this.array = array; this.op = op; this.lo = lo; this.hi = hi;
148  *   }
149  *
150  *   public void compute() { // version 1
151  *     if (hi - lo >= 2) {
152  *       int mid = (lo + hi) >>> 1;
153  *       setPendingCount(2); // must set pending count before fork
154  *       new ForEach(this, array, op, mid, hi).fork(); // right child
155  *       new ForEach(this, array, op, lo, mid).fork(); // left child
156  *     }
157  *     else if (hi > lo)
158  *       op.apply(array[lo]);
159  *     tryComplete();
160  *   }
161  * }}</pre>
162  *
163  * This design can be improved by noticing that in the recursive case,
164  * the task has nothing to do after forking its right task, so can
165  * directly invoke its left task before returning. (This is an analog
166  * of tail recursion removal.)  Also, because the task returns upon
167  * executing its left task (rather than falling through to invoke
168  * {@code tryComplete}) the pending count is set to one:
169  *
170  * <pre> {@code
171  * class ForEach<E> ...
172  *   public void compute() { // version 2
173  *     if (hi - lo >= 2) {
174  *       int mid = (lo + hi) >>> 1;
175  *       setPendingCount(1); // only one pending
176  *       new ForEach(this, array, op, mid, hi).fork(); // right child
177  *       new ForEach(this, array, op, lo, mid).compute(); // direct invoke
178  *     }
179  *     else {
180  *       if (hi > lo)
181  *         op.apply(array[lo]);
182  *       tryComplete();
183  *     }
184  *   }
185  * }</pre>
186  *
187  * As a further improvement, notice that the left task need not even exist.
188  * Instead of creating a new one, we can iterate using the original task,
189  * and add a pending count for each fork.  Additionally, because no task
190  * in this tree implements an {@link #onCompletion(CountedCompleter)} method,
191  * {@code tryComplete()} can be replaced with {@link #propagateCompletion}.
192  *
193  * <pre> {@code
194  * class ForEach<E> ...
195  *   public void compute() { // version 3
196  *     int l = lo,  h = hi;
197  *     while (h - l >= 2) {
198  *       int mid = (l + h) >>> 1;
199  *       addToPendingCount(1);
200  *       new ForEach(this, array, op, mid, h).fork(); // right child
201  *       h = mid;
202  *     }
203  *     if (h > l)
204  *       op.apply(array[l]);
205  *     propagateCompletion();
206  *   }
207  * }</pre>
208  *
209  * Additional improvements of such classes might entail precomputing
210  * pending counts so that they can be established in constructors,
211  * specializing classes for leaf steps, subdividing by say, four,
212  * instead of two per iteration, and using an adaptive threshold
213  * instead of always subdividing down to single elements.
214  *
215  * <p><b>Searching.</b> A tree of CountedCompleters can search for a
216  * value or property in different parts of a data structure, and
217  * report a result in an {@link
218  * java.util.concurrent.atomic.AtomicReference AtomicReference} as
219  * soon as one is found. The others can poll the result to avoid
220  * unnecessary work. (You could additionally {@linkplain #cancel
221  * cancel} other tasks, but it is usually simpler and more efficient
222  * to just let them notice that the result is set and if so skip
223  * further processing.)  Illustrating again with an array using full
224  * partitioning (again, in practice, leaf tasks will almost always
225  * process more than one element):
226  *
227  * <pre> {@code
228  * class Searcher<E> extends CountedCompleter<E> {
229  *   final E[] array; final AtomicReference<E> result; final int lo, hi;
230  *   Searcher(CountedCompleter<?> p, E[] array, AtomicReference<E> result, int lo, int hi) {
231  *     super(p);
232  *     this.array = array; this.result = result; this.lo = lo; this.hi = hi;
233  *   }
234  *   public E getRawResult() { return result.get(); }
235  *   public void compute() { // similar to ForEach version 3
236  *     int l = lo,  h = hi;
237  *     while (result.get() == null && h >= l) {
238  *       if (h - l >= 2) {
239  *         int mid = (l + h) >>> 1;
240  *         addToPendingCount(1);
241  *         new Searcher(this, array, result, mid, h).fork();
242  *         h = mid;
243  *       }
244  *       else {
245  *         E x = array[l];
246  *         if (matches(x) && result.compareAndSet(null, x))
247  *           quietlyCompleteRoot(); // root task is now joinable
248  *         break;
249  *       }
250  *     }
251  *     tryComplete(); // normally complete whether or not found
252  *   }
253  *   boolean matches(E e) { ... } // return true if found
254  *
255  *   public static <E> E search(E[] array) {
256  *       return new Searcher<E>(null, array, new AtomicReference<E>(), 0, array.length).invoke();
257  *   }
258  * }}</pre>
259  *
260  * In this example, as well as others in which tasks have no other
261  * effects except to compareAndSet a common result, the trailing
262  * unconditional invocation of {@code tryComplete} could be made
263  * conditional ({@code if (result.get() == null) tryComplete();})
264  * because no further bookkeeping is required to manage completions
265  * once the root task completes.
266  *
267  * <p><b>Recording subtasks.</b> CountedCompleter tasks that combine
268  * results of multiple subtasks usually need to access these results
269  * in method {@link #onCompletion(CountedCompleter)}. As illustrated in the following
270  * class (that performs a simplified form of map-reduce where mappings
271  * and reductions are all of type {@code E}), one way to do this in
272  * divide and conquer designs is to have each subtask record its
273  * sibling, so that it can be accessed in method {@code onCompletion}.
274  * This technique applies to reductions in which the order of
275  * combining left and right results does not matter; ordered
276  * reductions require explicit left/right designations.  Variants of
277  * other streamlinings seen in the above examples may also apply.
278  *
279  * <pre> {@code
280  * class MyMapper<E> { E apply(E v) {  ...  } }
281  * class MyReducer<E> { E apply(E x, E y) {  ...  } }
282  * class MapReducer<E> extends CountedCompleter<E> {
283  *   final E[] array; final MyMapper<E> mapper;
284  *   final MyReducer<E> reducer; final int lo, hi;
285  *   MapReducer<E> sibling;
286  *   E result;
287  *   MapReducer(CountedCompleter<?> p, E[] array, MyMapper<E> mapper,
288  *              MyReducer<E> reducer, int lo, int hi) {
289  *     super(p);
290  *     this.array = array; this.mapper = mapper;
291  *     this.reducer = reducer; this.lo = lo; this.hi = hi;
292  *   }
293  *   public void compute() {
294  *     if (hi - lo >= 2) {
295  *       int mid = (lo + hi) >>> 1;
296  *       MapReducer<E> left = new MapReducer(this, array, mapper, reducer, lo, mid);
297  *       MapReducer<E> right = new MapReducer(this, array, mapper, reducer, mid, hi);
298  *       left.sibling = right;
299  *       right.sibling = left;
300  *       setPendingCount(1); // only right is pending
301  *       right.fork();
302  *       left.compute();     // directly execute left
303  *     }
304  *     else {
305  *       if (hi > lo)
306  *           result = mapper.apply(array[lo]);
307  *       tryComplete();
308  *     }
309  *   }
310  *   public void onCompletion(CountedCompleter<?> caller) {
311  *     if (caller != this) {
312  *       MapReducer<E> child = (MapReducer<E>)caller;
313  *       MapReducer<E> sib = child.sibling;
314  *       if (sib == null || sib.result == null)
315  *         result = child.result;
316  *       else
317  *         result = reducer.apply(child.result, sib.result);
318  *     }
319  *   }
320  *   public E getRawResult() { return result; }
321  *
322  *   public static <E> E mapReduce(E[] array, MyMapper<E> mapper, MyReducer<E> reducer) {
323  *     return new MapReducer<E>(null, array, mapper, reducer,
324  *                              0, array.length).invoke();
325  *   }
326  * }}</pre>
327  *
328  * Here, method {@code onCompletion} takes a form common to many
329  * completion designs that combine results. This callback-style method
330  * is triggered once per task, in either of the two different contexts
331  * in which the pending count is, or becomes, zero: (1) by a task
332  * itself, if its pending count is zero upon invocation of {@code
333  * tryComplete}, or (2) by any of its subtasks when they complete and
334  * decrement the pending count to zero. The {@code caller} argument
335  * distinguishes cases.  Most often, when the caller is {@code this},
336  * no action is necessary. Otherwise the caller argument can be used
337  * (usually via a cast) to supply a value (and/or links to other
338  * values) to be combined.  Assuming proper use of pending counts, the
339  * actions inside {@code onCompletion} occur (once) upon completion of
340  * a task and its subtasks. No additional synchronization is required
341  * within this method to ensure thread safety of accesses to fields of
342  * this task or other completed tasks.
343  *
344  * <p><b>Completion Traversals</b>. If using {@code onCompletion} to
345  * process completions is inapplicable or inconvenient, you can use
346  * methods {@link #firstComplete} and {@link #nextComplete} to create
347  * custom traversals.  For example, to define a MapReducer that only
348  * splits out right-hand tasks in the form of the third ForEach
349  * example, the completions must cooperatively reduce along
350  * unexhausted subtask links, which can be done as follows:
351  *
352  * <pre> {@code
353  * class MapReducer<E> extends CountedCompleter<E> { // version 2
354  *   final E[] array; final MyMapper<E> mapper;
355  *   final MyReducer<E> reducer; final int lo, hi;
356  *   MapReducer<E> forks, next; // record subtask forks in list
357  *   E result;
358  *   MapReducer(CountedCompleter<?> p, E[] array, MyMapper<E> mapper,
359  *              MyReducer<E> reducer, int lo, int hi, MapReducer<E> next) {
360  *     super(p);
361  *     this.array = array; this.mapper = mapper;
362  *     this.reducer = reducer; this.lo = lo; this.hi = hi;
363  *     this.next = next;
364  *   }
365  *   public void compute() {
366  *     int l = lo,  h = hi;
367  *     while (h - l >= 2) {
368  *       int mid = (l + h) >>> 1;
369  *       addToPendingCount(1);
370  *       (forks = new MapReducer(this, array, mapper, reducer, mid, h, forks)).fork();
371  *       h = mid;
372  *     }
373  *     if (h > l)
374  *       result = mapper.apply(array[l]);
375  *     // process completions by reducing along and advancing subtask links
376  *     for (CountedCompleter<?> c = firstComplete(); c != null; c = c.nextComplete()) {
377  *       for (MapReducer t = (MapReducer)c, s = t.forks;  s != null; s = t.forks = s.next)
378  *         t.result = reducer.apply(t.result, s.result);
379  *     }
380  *   }
381  *   public E getRawResult() { return result; }
382  *
383  *   public static <E> E mapReduce(E[] array, MyMapper<E> mapper, MyReducer<E> reducer) {
384  *     return new MapReducer<E>(null, array, mapper, reducer,
385  *                              0, array.length, null).invoke();
386  *   }
387  * }}</pre>
388  *
389  * <p><b>Triggers.</b> Some CountedCompleters are themselves never
390  * forked, but instead serve as bits of plumbing in other designs;
391  * including those in which the completion of one or more async tasks
392  * triggers another async task. For example:
393  *
394  * <pre> {@code
395  * class HeaderBuilder extends CountedCompleter<...> { ... }
396  * class BodyBuilder extends CountedCompleter<...> { ... }
397  * class PacketSender extends CountedCompleter<...> {
398  *   PacketSender(...) { super(null, 1); ... } // trigger on second completion
399  *   public void compute() { } // never called
400  *   public void onCompletion(CountedCompleter<?> caller) { sendPacket(); }
401  * }
402  * // sample use:
403  * PacketSender p = new PacketSender();
404  * new HeaderBuilder(p, ...).fork();
405  * new BodyBuilder(p, ...).fork();
406  * }</pre>
407  *
408  * @since 1.8
409  * @author Doug Lea
410  */
411 public abstract class CountedCompleter<T> extends ForkJoinTask<T> {
412     private static final long serialVersionUID = 5232453752276485070L;
413 
414     /** This task's completer, or null if none */
415     final CountedCompleter<?> completer;
416     /** The number of pending tasks until completion */
417     volatile int pending;
418 
419     /**
420      * Creates a new CountedCompleter with the given completer
421      * and initial pending count.
422      *
423      * @param completer this task's completer, or {@code null} if none
424      * @param initialPendingCount the initial pending count
425      */
426     protected CountedCompleter(CountedCompleter<?> completer,
427                                int initialPendingCount) {
428         this.completer = completer;
429         this.pending = initialPendingCount;
430     }
431 
432     /**
433      * Creates a new CountedCompleter with the given completer
434      * and an initial pending count of zero.
435      *
436      * @param completer this task's completer, or {@code null} if none
437      */
438     protected CountedCompleter(CountedCompleter<?> completer) {
439         this.completer = completer;
440     }
441 
442     /**
443      * Creates a new CountedCompleter with no completer
444      * and an initial pending count of zero.
445      */
446     protected CountedCompleter() {
447         this.completer = null;
448     }
449 
450     /**
451      * The main computation performed by this task.
452      */
453     public abstract void compute();
454 
455     /**
456      * Performs an action when method {@link #tryComplete} is invoked
457      * and the pending count is zero, or when the unconditional
458      * method {@link #complete} is invoked.  By default, this method
459      * does nothing. You can distinguish cases by checking the
460      * identity of the given caller argument. If not equal to {@code
461      * this}, then it is typically a subtask that may contain results
462      * (and/or links to other results) to combine.
463      *
464      * @param caller the task invoking this method (which may
465      * be this task itself)
466      */
467     public void onCompletion(CountedCompleter<?> caller) {
468     }
469 
470     /**
471      * Performs an action when method {@link
472      * #completeExceptionally(Throwable)} is invoked or method {@link
473      * #compute} throws an exception, and this task has not already
474      * otherwise completed normally. On entry to this method, this task
475      * {@link ForkJoinTask#isCompletedAbnormally}.  The return value
476      * of this method controls further propagation: If {@code true}
477      * and this task has a completer that has not completed, then that
478      * completer is also completed exceptionally, with the same
479      * exception as this completer.  The default implementation of
480      * this method does nothing except return {@code true}.
481      *
482      * @param ex the exception
483      * @param caller the task invoking this method (which may
484      * be this task itself)
485      * @return {@code true} if this exception should be propagated to this
486      * task's completer, if one exists
487      */
488     public boolean onExceptionalCompletion(Throwable ex, CountedCompleter<?> caller) {
489         return true;
490     }
491 
492     /**
493      * Returns the completer established in this task's constructor,
494      * or {@code null} if none.
495      *
496      * @return the completer
497      */
498     public final CountedCompleter<?> getCompleter() {
499         return completer;
500     }
501 
502     /**
503      * Returns the current pending count.
504      *
505      * @return the current pending count
506      */
507     public final int getPendingCount() {
508         return pending;
509     }
510 
511     /**
512      * Sets the pending count to the given value.
513      *
514      * @param count the count
515      */
516     public final void setPendingCount(int count) {
517         pending = count;
518     }
519 
520     /**
521      * Adds (atomically) the given value to the pending count.
522      *
523      * @param delta the value to add
524      */
525     public final void addToPendingCount(int delta) {
526         U.getAndAddInt(this, PENDING, delta);
527     }
528 
529     /**
530      * Sets (atomically) the pending count to the given count only if
531      * it currently holds the given expected value.
532      *
533      * @param expected the expected value
534      * @param count the new value
535      * @return {@code true} if successful
536      */
537     public final boolean compareAndSetPendingCount(int expected, int count) {
538         return U.compareAndSwapInt(this, PENDING, expected, count);
539     }
540 
541     /**
542      * If the pending count is nonzero, (atomically) decrements it.
543      *
544      * @return the initial (undecremented) pending count holding on entry
545      * to this method
546      */
547     public final int decrementPendingCountUnlessZero() {
548         int c;
549         do {} while ((c = pending) != 0 &&
550                      !U.compareAndSwapInt(this, PENDING, c, c - 1));
551         return c;
552     }
553 
554     /**
555      * Returns the root of the current computation; i.e., this
556      * task if it has no completer, else its completer's root.
557      *
558      * @return the root of the current computation
559      */
560     public final CountedCompleter<?> getRoot() {
561         CountedCompleter<?> a = this, p;
562         while ((p = a.completer) != null)
563             a = p;
564         return a;
565     }
566 
567     /**
568      * If the pending count is nonzero, decrements the count;
569      * otherwise invokes {@link #onCompletion(CountedCompleter)}
570      * and then similarly tries to complete this task's completer,
571      * if one exists, else marks this task as complete.
572      */
573     public final void tryComplete() {
574         CountedCompleter<?> a = this, s = a;
575         for (int c;;) {
576             if ((c = a.pending) == 0) {
577                 a.onCompletion(s);
578                 if ((a = (s = a).completer) == null) {
579                     s.quietlyComplete();
580                     return;
581                 }
582             }
583             else if (U.compareAndSwapInt(a, PENDING, c, c - 1))
584                 return;
585         }
586     }

588     /**
589      * Equivalent to {@link #tryComplete} but does not invoke {@link
590      * #onCompletion(CountedCompleter)} along the completion path:
591      * If the pending count is nonzero, decrements the count;
592      * otherwise, similarly tries to complete this task's completer, if
593      * one exists, else marks this task as complete. This method may be
594      * useful in cases where {@code onCompletion} should not, or need
595      * not, be invoked for each completer in a computation.
596      */
597     public final void propagateCompletion() {
598         CountedCompleter<?> a = this, s = a;
599         for (int c;;) {
600             if ((c = a.pending) == 0) {
601                 if ((a = (s = a).completer) == null) {
602                     s.quietlyComplete();
603                     return;
604                 }
605             }
606             else if (U.compareAndSwapInt(a, PENDING, c, c - 1))
607                 return;
608         }
609     }

611     /**
612      * Regardless of pending count, invokes
613      * {@link #onCompletion(CountedCompleter)}, marks this task as
614      * complete and further triggers {@link #tryComplete} on this
615      * task's completer, if one exists.  The given rawResult is
616      * used as an argument to {@link #setRawResult} before invoking
617      * {@link #onCompletion(CountedCompleter)} or marking this task
618      * as complete; its value is meaningful only for classes
619      * overriding {@code setRawResult}.  This method does not modify
620      * the pending count.
621      *
622      * <p>This method may be useful when forcing completion as soon as
623      * any one (versus all) of several subtask results are obtained.
624      * However, in the common (and recommended) case in which {@code
625      * setRawResult} is not overridden, this effect can be obtained
626      * more simply using {@code quietlyCompleteRoot();}.
627      *
628      * @param rawResult the raw result
629      */
630     public void complete(T rawResult) {
631         CountedCompleter<?> p;
632         setRawResult(rawResult);
633         onCompletion(this);
634         quietlyComplete();
635         if ((p = completer) != null)
636             p.tryComplete();
637     }

639     /**
640      * If this task's pending count is zero, returns this task;
641      * otherwise decrements its pending count and returns {@code
642      * null}. This method is designed to be used with {@link
643      * #nextComplete} in completion traversal loops.
644      *
645      * @return this task, if pending count was zero, else {@code null}
646      */
647     public final CountedCompleter<?> firstComplete() {
648         for (int c;;) {
649             if ((c = pending) == 0)
650                 return this;
651             else if (U.compareAndSwapInt(this, PENDING, c, c - 1))
652                 return null;
653         }
654     }

656     /**
657      * If this task does not have a completer, invokes {@link
658      * ForkJoinTask#quietlyComplete} and returns {@code null}.  Or, if
659      * the completer's pending count is non-zero, decrements that
660      * pending count and returns {@code null}.  Otherwise, returns the
661      * completer.  This method can be used as part of a completion
662      * traversal loop for homogeneous task hierarchies:
663      *
664      * <pre> {@code
665      * for (CountedCompleter<?> c = firstComplete();
666      *      c != null;
667      *      c = c.nextComplete()) {
668      *   // ... process c ...
669      * }}</pre>
670      *
671      * @return the completer, or {@code null} if none
672      */
673     public final CountedCompleter<?> nextComplete() {
674         CountedCompleter<?> p;
675         if ((p = completer) != null)
676             return p.firstComplete();
677         else {
678             quietlyComplete();
679             return null;
680         }
681     }

683     /**
684      * Equivalent to {@code getRoot().quietlyComplete()}.
685      */
686     public final void quietlyCompleteRoot() {
687         for (CountedCompleter<?> a = this, p;;) {
688             if ((p = a.completer) == null) {
689                 a.quietlyComplete();
690                 return;
691             }
692             a = p;
693         }
694     }

696     /**
697      * If this task has not completed, attempts to process at most the
698      * given number of other unprocessed tasks for which this task is
699      * on the completion path, if any are known to exist.
700      *
701      * @param maxTasks the maximum number of tasks to process.  If
702      *                 less than or equal to zero, then no tasks are
703      *                 processed.
704      */
705     public final void helpComplete(int maxTasks) {
706         Thread t; ForkJoinWorkerThread wt;
707         if (maxTasks > 0 && status >= 0) {
708             if ((t = Thread.currentThread()) instanceof ForkJoinWorkerThread)
709                 (wt = (ForkJoinWorkerThread)t).pool.
710                     helpComplete(wt.workQueue, this, maxTasks);
711             else
712                 ForkJoinPool.common.externalHelpComplete(this, maxTasks);
713         }
714     }

716     /**
717      * Supports ForkJoinTask exception propagation.
718      */
719     void internalPropagateException(Throwable ex) {
720         CountedCompleter<?> a = this, s = a;
721         while (a.onExceptionalCompletion(ex, s) &&
722                (a = (s = a).completer) != null && a.status >= 0 &&
723                a.recordExceptionalCompletion(ex) == EXCEPTIONAL)
724             ;
725     }
