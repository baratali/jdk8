94  /**
95   * Localized definitions of the day-of-week, week-of-month and week-of-year fields.
96   * <p>
97   * A standard week is seven days long, but cultures have different definitions for some
98   * other aspects of a week. This class represents the definition of the week, for the
99   * purpose of providing {@link TemporalField} instances.
100  * <p>
101  * WeekFields provides five fields,
102  * {@link #dayOfWeek()}, {@link #weekOfMonth()}, {@link #weekOfYear()},
103  * {@link #weekOfWeekBasedYear()}, and {@link #weekBasedYear()}
104  * that provide access to the values from any {@linkplain Temporal temporal object}.
105  * <p>
106  * The computations for day-of-week, week-of-month, and week-of-year are based
107  * on the  {@linkplain ChronoField#YEAR proleptic-year},
108  * {@linkplain ChronoField#MONTH_OF_YEAR month-of-year},
109  * {@linkplain ChronoField#DAY_OF_MONTH day-of-month}, and
110  * {@linkplain ChronoField#DAY_OF_WEEK ISO day-of-week} which are based on the
111  * {@linkplain ChronoField#EPOCH_DAY epoch-day} and the chronology.
112  * The values may not be aligned with the {@linkplain ChronoField#YEAR_OF_ERA year-of-Era}
113  * depending on the Chronology.
114  * <p>A week is defined by:
115  * <ul>
116  * <li>The first day-of-week.
117  * For example, the ISO-8601 standard considers Monday to be the first day-of-week.
118  * <li>The minimal number of days in the first week.
119  * For example, the ISO-8601 standard counts the first week as needing at least 4 days.
120  * </ul>
121  * Together these two values allow a year or month to be divided into weeks.
122  *
123  * <h3>Week of Month</h3>
124  * One field is used: week-of-month.
125  * The calculation ensures that weeks never overlap a month boundary.
126  * The month is divided into periods where each period starts on the defined first day-of-week.
127  * The earliest period is referred to as week 0 if it has less than the minimal number of days
128  * and week 1 if it has at least the minimal number of days.
129  *
130  * <table cellpadding="0" cellspacing="3" border="0" style="text-align: left; width: 50%;">
131  * <caption>Examples of WeekFields</caption>
132  * <tr><th>Date</th><td>Day-of-week</td>
133  *  <td>First day: Monday<br>Minimal days: 4</td><td>First day: Monday<br>Minimal days: 5</td></tr>
134  * <tr><th>2008-12-31</th><td>Wednesday</td>
135  *  <td>Week 5 of December 2008</td><td>Week 5 of December 2008</td></tr>
136  * <tr><th>2009-01-01</th><td>Thursday</td>
137  *  <td>Week 1 of January 2009</td><td>Week 0 of January 2009</td></tr>
138  * <tr><th>2009-01-04</th><td>Sunday</td>
139  *  <td>Week 1 of January 2009</td><td>Week 0 of January 2009</td></tr>
140  * <tr><th>2009-01-05</th><td>Monday</td>
141  *  <td>Week 2 of January 2009</td><td>Week 1 of January 2009</td></tr>
142  * </table>
143  *
144  * <h3>Week of Year</h3>
145  * One field is used: week-of-year.
146  * The calculation ensures that weeks never overlap a year boundary.
147  * The year is divided into periods where each period starts on the defined first day-of-week.
148  * The earliest period is referred to as week 0 if it has less than the minimal number of days
149  * and week 1 if it has at least the minimal number of days.
150  *
151  * <h3>Week Based Year</h3>
152  * Two fields are used for week-based-year, one for the
153  * {@link #weekOfWeekBasedYear() week-of-week-based-year} and one for
154  * {@link #weekBasedYear() week-based-year}.  In a week-based-year, each week
155  * belongs to only a single year.  Week 1 of a year is the first week that
156  * starts on the first day-of-week and has at least the minimum number of days.
157  * The first and last weeks of a year may contain days from the
158  * previous calendar year or next calendar year respectively.
159  *
160  * <table cellpadding="0" cellspacing="3" border="0" style="text-align: left; width: 50%;">
161  * <caption>Examples of WeekFields for week-based-year</caption>
162  * <tr><th>Date</th><td>Day-of-week</td>
163  *  <td>First day: Monday<br>Minimal days: 4</td><td>First day: Monday<br>Minimal days: 5</td></tr>
164  * <tr><th>2008-12-31</th><td>Wednesday</td>
165  *  <td>Week 1 of 2009</td><td>Week 53 of 2008</td></tr>
166  * <tr><th>2009-01-01</th><td>Thursday</td>
167  *  <td>Week 1 of 2009</td><td>Week 53 of 2008</td></tr>
168  * <tr><th>2009-01-04</th><td>Sunday</td>
169  *  <td>Week 1 of 2009</td><td>Week 53 of 2008</td></tr>
170  * <tr><th>2009-01-05</th><td>Monday</td>
171  *  <td>Week 2 of 2009</td><td>Week 1 of 2009</td></tr>
172  * </table>
173  *
174  * @implSpec
175  * This class is immutable and thread-safe.
176  *
177  * @since 1.8
178  */

184     // allow week-of-year outer range [0 to 54]

195      * has a minimum of 4 days.
196      * <p>
197      * The ISO-8601 standard defines a calendar system based on weeks.
198      * It uses the week-based-year and week-of-week-based-year concepts to split
199      * up the passage of days instead of the standard year/month/day.
200      * <p>
201      * Note that the first week may start in the previous calendar year.
202      * Note also that the first few days of a calendar year may be in the
203      * week-based-year corresponding to the previous calendar year.
204      */
205     public static final WeekFields ISO = new WeekFields(DayOfWeek.MONDAY, 4);

211      * Defined as starting on Sunday and with a minimum of 1 day in the month.
212      * This week definition is in use in the US and other European countries.
213      */
214     public static final WeekFields SUNDAY_START = WeekFields.of(DayOfWeek.SUNDAY, 1);

216     /**
217      * The unit that represents week-based-years for the purpose of addition and subtraction.
218      * <p>
219      * This allows a number of week-based-years to be added to, or subtracted from, a date.
220      * The unit is equal to either 52 or 53 weeks.
221      * The estimated duration of a week-based-year is the same as that of a standard ISO
222      * year at {@code 365.2425 Days}.
223      * <p>
224      * The rules for addition add the number of week-based-years to the existing value
225      * for the week-based-year field retaining the week-of-week-based-year
226      * and day-of-week, unless the week number it too large for the target year.
227      * In that case, the week is set to the last week of the year
228      * with the same day-of-week.
229      * <p>
230      * This unit is an immutable and thread-safe singleton.
231      */
232     public static final TemporalUnit WEEK_BASED_YEARS = IsoFields.WEEK_BASED_YEARS;

277     /**
278      * Obtains an instance of {@code WeekFields} appropriate for a locale.
279      * <p>
280      * This will look up appropriate values from the provider of localization data.
281      *
282      * @param locale  the locale to use, not null
283      * @return the week-definition, not null
284      */
285     public static WeekFields of(Locale locale) {
286         Objects.requireNonNull(locale, "locale");
287         locale = new Locale(locale.getLanguage(), locale.getCountry());  // elminate variants
288 
289         int calDow = CalendarDataUtility.retrieveFirstDayOfWeek(locale);
290         DayOfWeek dow = DayOfWeek.SUNDAY.plus(calDow - 1);
291         int minDays = CalendarDataUtility.retrieveMinimalDaysInFirstWeek(locale);
292         return WeekFields.of(dow, minDays);
293     }

295     /**
296      * Obtains an instance of {@code WeekFields} from the first day-of-week and minimal days.
297      * <p>
298      * The first day-of-week defines the ISO {@code DayOfWeek} that is day 1 of the week.
299      * The minimal number of days in the first week defines how many days must be present
300      * in a month or year, starting from the first day-of-week, before the week is counted
301      * as the first week. A value of 1 will count the first day of the month or year as part
302      * of the first week, whereas a value of 7 will require the whole seven days to be in
303      * the new month or year.
304      * <p>
305      * WeekFields instances are singletons; for each unique combination
306      * of {@code firstDayOfWeek} and {@code minimalDaysInFirstWeek} the
307      * the same instance will be returned.
308      *
309      * @param firstDayOfWeek  the first day of the week, not null
310      * @param minimalDaysInFirstWeek  the minimal number of days in the first week, from 1 to 7
311      * @return the week-definition, not null
312      * @throws IllegalArgumentException if the minimal days value is less than one
313      *      or greater than 7
314      */
315     public static WeekFields of(DayOfWeek firstDayOfWeek, int minimalDaysInFirstWeek) {
316         String key = firstDayOfWeek.toString() + minimalDaysInFirstWeek;
317         WeekFields rules = CACHE.get(key);
318         if (rules == null) {
319             rules = new WeekFields(firstDayOfWeek, minimalDaysInFirstWeek);
320             CACHE.putIfAbsent(key, rules);
321             rules = CACHE.get(key);
322         }
323         return rules;
324     }

326     //-----------------------------------------------------------------------
327     /**
328      * Creates an instance of the definition.
329      *
330      * @param firstDayOfWeek  the first day of the week, not null
331      * @param minimalDaysInFirstWeek  the minimal number of days in the first week, from 1 to 7
332      * @throws IllegalArgumentException if the minimal days value is invalid
333      */
334     private WeekFields(DayOfWeek firstDayOfWeek, int minimalDaysInFirstWeek) {
335         Objects.requireNonNull(firstDayOfWeek, "firstDayOfWeek");
336         if (minimalDaysInFirstWeek < 1 || minimalDaysInFirstWeek > 7) {
337             throw new IllegalArgumentException("Minimal number of days is invalid");
338         }
339         this.firstDayOfWeek = firstDayOfWeek;
340         this.minimalDays = minimalDaysInFirstWeek;
341     }

346      * Check that the values are valid.
347      * @throws InvalidObjectException if the serialized object has an invalid

353         s.defaultReadObject();

355             throw new InvalidObjectException("firstDayOfWeek is null");
356         }
357 
358         if (minimalDays < 1 || minimalDays > 7) {
359             throw new InvalidObjectException("Minimal number of days is invalid");
360         }
361     }

429     /**
430      * Returns a field to access the week of month based on this {@code WeekFields}.
431      * <p>
432      * This represents the concept of the count of weeks within the month where weeks
433      * start on a fixed day-of-week, such as Monday.
434      * This field is typically used with {@link WeekFields#dayOfWeek()}.
435      * <p>
436      * Week one (1) is the week starting on the {@link WeekFields#getFirstDayOfWeek}
437      * where there are at least {@link WeekFields#getMinimalDaysInFirstWeek()} days in the month.
438      * Thus, week one may start up to {@code minDays} days before the start of the month.
439      * If the first week starts after the start of the month then the period before is week zero (0).
440      * <p>
441      * For example:<br>
442      * - if the 1st day of the month is a Monday, week one starts on the 1st and there is no week zero<br>
443      * - if the 2nd day of the month is a Monday, week one starts on the 2nd and the 1st is in week zero<br>
444      * - if the 4th day of the month is a Monday, week one starts on the 4th and the 1st to 3rd is in week zero<br>
445      * - if the 5th day of the month is a Monday, week two starts on the 5th and the 1st to 4th is in week one<br>
446      * <p>
447      * This field can be used with any calendar system.
448      * <p>
449      * In the resolving phase of parsing, a date can be created from a year,
450      * week-of-month, month-of-year and day-of-week.
451      * <p>
452      * In {@linkplain ResolverStyle#STRICT strict mode}, all four fields are
453      * validated against their range of valid values. The week-of-month field
454      * is validated to ensure that the resulting month is the month requested.
455      * <p>
456      * In {@linkplain ResolverStyle#SMART smart mode}, all four fields are
457      * validated against their range of valid values. The week-of-month field
458      * is validated from 0 to 6, meaning that the resulting date can be in a
459      * different month to that specified.
460      * <p>
461      * In {@linkplain ResolverStyle#LENIENT lenient mode}, the year and day-of-week
462      * are validated against the range of valid values. The resulting date is calculated
463      * equivalent to the following four stage approach.
464      * First, create a date on the first day of the first week of January in the requested year.
465      * Then take the month-of-year, subtract one, and add the amount in months to the date.
466      * Then take the week-of-month, subtract one, and add the amount in weeks to the date.
467      * Finally, adjust to the correct day-of-week within the localized week.
468      *
469      * @return a field providing access to the week-of-month, not null
470      */
471     public TemporalField weekOfMonth() {
472         return weekOfMonth;
473     }

520     /**
521      * Returns a field to access the week of a week-based-year based on this {@code WeekFields}.
522      * <p>
523      * This represents the concept of the count of weeks within the year where weeks
524      * start on a fixed day-of-week, such as Monday and each week belongs to exactly one year.
525      * This field is typically used with {@link WeekFields#dayOfWeek()} and
526      * {@link WeekFields#weekBasedYear()}.
527      * <p>
528      * Week one(1) is the week starting on the {@link WeekFields#getFirstDayOfWeek}
529      * where there are at least {@link WeekFields#getMinimalDaysInFirstWeek()} days in the year.
530      * If the first week starts after the start of the year then the period before
531      * is in the last week of the previous year.
532      * <p>
533      * For example:<br>
534      * - if the 1st day of the year is a Monday, week one starts on the 1st<br>
535      * - if the 2nd day of the year is a Monday, week one starts on the 2nd and
536      *   the 1st is in the last week of the previous year<br>
537      * - if the 4th day of the year is a Monday, week one starts on the 4th and
538      *   the 1st to 3rd is in the last week of the previous year<br>
539      * - if the 5th day of the year is a Monday, week two starts on the 5th and
540      *   the 1st to 4th is in week one<br>
541      * <p>
542      * This field can be used with any calendar system.
543      * <p>
544      * In the resolving phase of parsing, a date can be created from a week-based-year,
545      * week-of-year and day-of-week.
546      * <p>
547      * In {@linkplain ResolverStyle#STRICT strict mode}, all three fields are
548      * validated against their range of valid values. The week-of-year field
549      * is validated to ensure that the resulting week-based-year is the
550      * week-based-year requested.
551      * <p>
552      * In {@linkplain ResolverStyle#SMART smart mode}, all three fields are
553      * validated against their range of valid values. The week-of-week-based-year field
554      * is validated from 1 to 53, meaning that the resulting date can be in the
555      * following week-based-year to that specified.
556      * <p>
557      * In {@linkplain ResolverStyle#LENIENT lenient mode}, the year and day-of-week
558      * are validated against the range of valid values. The resulting date is calculated
559      * equivalent to the following three stage approach.
560      * First, create a date on the first day of the first week in the requested week-based-year.
561      * Then take the week-of-week-based-year, subtract one, and add the amount in weeks to the date.
562      * Finally, adjust to the correct day-of-week within the localized week.
563      *
564      * @return a field providing access to the week-of-week-based-year, not null
565      */
566     public TemporalField weekOfWeekBasedYear() {
567         return weekOfWeekBasedYear;
568     }

722                 int yowby, int wowby, int dow) {
723             ChronoLocalDate date = chrono.date(yowby, 1, 1);
724             int ldow = localizedDayOfWeek(date);

796         /**
797          * Returns the year of week-based-year for the temporal.
798          * The year can be the previous year, the current year, or the next year.
799          * @param temporal a date of any chronology, not null
800          * @return the year of week-based-year for the date
801          */
802         private int localizedWeekBasedYear(TemporalAccessor temporal) {
803             int dow = localizedDayOfWeek(temporal);
804             int year = temporal.get(YEAR);
805             int doy = temporal.get(DAY_OF_YEAR);
806             int offset = startOfWeekOffset(doy, dow);
807             int week = computeWeek(offset, doy);
808             if (week == 0) {
809                 // Day is in end of week of previous year; return the previous year
810                 return year - 1;
811             } else {
812                 // If getting close to end of year, use higher precision logic
813                 // Check if date of year is in partial week associated with next year
814                 ValueRange dayRange = temporal.range(DAY_OF_YEAR);
815                 int yearLen = (int)dayRange.getMaximum();
816                 int newYearWeek = computeWeek(offset, yearLen + weekDef.getMinimalDaysInFirstWeek());
817                 if (week >= newYearWeek) {
818                     return year + 1;
819                 }
820             }
821             return year;
822         }

824         /**
825          * Returns the week of week-based-year for the temporal.
826          * The week can be part of the previous year, the current year,
827          * or the next year depending on the week start and minimum number
828          * of days.
829          * @param temporal  a date of any chronology
830          * @return the week of the year
831          * @see #localizedWeekBasedYear(java.time.temporal.TemporalAccessor)
832          */
833         private int localizedWeekOfWeekBasedYear(TemporalAccessor temporal) {
834             int dow = localizedDayOfWeek(temporal);
835             int doy = temporal.get(DAY_OF_YEAR);
836             int offset = startOfWeekOffset(doy, dow);
837             int week = computeWeek(offset, doy);
838             if (week == 0) {
839                 // Day is in end of week of previous year
840                 // Recompute from the last day of the previous year
841                 ChronoLocalDate date = Chronology.from(temporal).date(temporal);
842                 date = date.minus(doy, DAYS);   // Back down into previous year
843                 return localizedWeekOfWeekBasedYear(date);
844             } else if (week > 50) {
845                 // If getting close to end of year, use higher precision logic
846                 // Check if date of year is in partial week associated with next year
847                 ValueRange dayRange = temporal.range(DAY_OF_YEAR);
848                 int yearLen = (int)dayRange.getMaximum();
849                 int newYearWeek = computeWeek(offset, yearLen + weekDef.getMinimalDaysInFirstWeek());
850                 if (week >= newYearWeek) {
851                     // Overlaps with week of following year; reduce to week in following year
852                     week = week - newYearWeek + 1;
853                 }
854             }
855             return week;
856         }

858         /**
859          * Returns an offset to align week start with a day of month or day of year.
860          *
861          * @param day  the day; 1 through infinity
862          * @param dow  the day of the week of that day; 1 through 7
863          * @return  an offset in days to align a day with the start of the first 'full' week
864          */
865         private int startOfWeekOffset(int day, int dow) {
866             // offset of first day corresponding to the day of week in first 7 days (zero origin)
867             int weekStart = Math.floorMod(day - dow, 7);
868             int offset = -weekStart;
869             if (weekStart + 1 > weekDef.getMinimalDaysInFirstWeek()) {
870                 // The previous week has the minimum days in the current month to be a 'week'
871                 offset = 7 - weekStart;
872             }
873             return offset;
874         }
875 
876         /**
877          * Returns the week number computed from the reference day and reference dayOfWeek.
878          *
879          * @param offset the offset to align a date with the start of week
880          *     from {@link #startOfWeekOffset}.
881          * @param day  the day for which to compute the week number
882          * @return the week number where zero is used for a partial week and 1 for the first full week
883          */
884         private int computeWeek(int offset, int day) {
885             return ((7 + offset + (day - 1)) / 7);
886         }

888         @SuppressWarnings("unchecked")
889         @Override
890         public <R extends Temporal> R adjustInto(R temporal, long newValue) {
891             // Check the new value and get the old value of the field
892             int newVal = range.checkValidIntValue(newValue, this);  // lenient check range
893             int currentVal = temporal.get(this);
894             if (newVal == currentVal) {
895                 return temporal;
896             }
897 
898             if (rangeUnit == FOREVER) {     // replace year of WeekBasedYear
899                 // Create a new date object with the same chronology,
900                 // the desired year and the same week and dow.
901                 int idow = temporal.get(weekDef.dayOfWeek);
902                 int wowby = temporal.get(weekDef.weekOfWeekBasedYear);
903                 return (R) ofWeekBasedYear(Chronology.from(temporal), (int)newValue, wowby, idow);
904             } else {
905                 // Compute the difference and add that using the base unit of the field
906                 return (R) temporal.plus(newVal - currentVal, baseUnit);
907             }
908         }

910         @Override
911         public ChronoLocalDate resolve(
912                 Map<TemporalField, Long> fieldValues, TemporalAccessor partialTemporal, ResolverStyle resolverStyle) {
913             final long value = fieldValues.get(this);
914             final int newValue = Math.toIntExact(value);  // broad limit makes overflow checking lighter
915             // first convert localized day-of-week to ISO day-of-week
916             // doing this first handles case where both ISO and localized were parsed and might mismatch
917             // day-of-week is always strict as two different day-of-week values makes lenient complex
918             if (rangeUnit == WEEKS) {  // day-of-week
919                 final int checkedValue = range.checkValidIntValue(value, this);  // no leniency as too complex
920                 final int startDow = weekDef.getFirstDayOfWeek().getValue();
921                 long isoDow = Math.floorMod((startDow - 1) + (checkedValue - 1), 7) + 1;
922                 fieldValues.remove(this);
923                 fieldValues.put(DAY_OF_WEEK, isoDow);
924                 return null;
925             }
926 
927             // can only build date if ISO day-of-week is present
928             if (fieldValues.containsKey(DAY_OF_WEEK) == false) {
929                 return null;
930             }
931             int isoDow = DAY_OF_WEEK.checkValidIntValue(fieldValues.get(DAY_OF_WEEK));
932             int dow = localizedDayOfWeek(isoDow);
933 
934             // build date
935             Chronology chrono = Chronology.from(partialTemporal);
936             if (fieldValues.containsKey(YEAR)) {
937                 int year = YEAR.checkValidIntValue(fieldValues.get(YEAR));  // validate
938                 if (rangeUnit == MONTHS && fieldValues.containsKey(MONTH_OF_YEAR)) {  // week-of-month
939                     long month = fieldValues.get(MONTH_OF_YEAR);  // not validated yet
940                     return resolveWoM(fieldValues, chrono, year, month, newValue, dow, resolverStyle);
941                 }
942                 if (rangeUnit == YEARS) {  // week-of-year
943                     return resolveWoY(fieldValues, chrono, year, newValue, dow, resolverStyle);
944                 }
945             } else if ((rangeUnit == WEEK_BASED_YEARS || rangeUnit == FOREVER) &&
946                     fieldValues.containsKey(weekDef.weekBasedYear) &&
947                     fieldValues.containsKey(weekDef.weekOfWeekBasedYear)) { // week-of-week-based-year and year-of-week-based-year
948                 return resolveWBY(fieldValues, chrono, dow, resolverStyle);
949             }
950             return null;
951         }

953         private ChronoLocalDate resolveWoM(
954                 Map<TemporalField, Long> fieldValues, Chronology chrono, int year, long month, long wom, int localDow, ResolverStyle resolverStyle) {
955             ChronoLocalDate date;
956             if (resolverStyle == ResolverStyle.LENIENT) {
957                 date = chrono.date(year, 1, 1).plus(Math.subtractExact(month, 1), MONTHS);
958                 long weeks = Math.subtractExact(wom, localizedWeekOfMonth(date));
959                 int days = localDow - localizedDayOfWeek(date);  // safe from overflow
960                 date = date.plus(Math.addExact(Math.multiplyExact(weeks, 7), days), DAYS);
961             } else {
962                 int monthValid = MONTH_OF_YEAR.checkValidIntValue(month);  // validate
963                 date = chrono.date(year, monthValid, 1);
964                 int womInt = range.checkValidIntValue(wom, this);  // validate
965                 int weeks = (int) (womInt - localizedWeekOfMonth(date));  // safe from overflow
966                 int days = localDow - localizedDayOfWeek(date);  // safe from overflow
967                 date = date.plus(weeks * 7 + days, DAYS);
968                 if (resolverStyle == ResolverStyle.STRICT && date.getLong(MONTH_OF_YEAR) != month) {
969                     throw new DateTimeException("Strict mode rejected resolved date as it is in a different month");
970                 }
971             }
972             fieldValues.remove(this);
973             fieldValues.remove(YEAR);
974             fieldValues.remove(MONTH_OF_YEAR);
975             fieldValues.remove(DAY_OF_WEEK);
976             return date;
977         }

979         private ChronoLocalDate resolveWoY(
980                 Map<TemporalField, Long> fieldValues, Chronology chrono, int year, long woy, int localDow, ResolverStyle resolverStyle) {
981             ChronoLocalDate date = chrono.date(year, 1, 1);
982             if (resolverStyle == ResolverStyle.LENIENT) {
983                 long weeks = Math.subtractExact(woy, localizedWeekOfYear(date));
984                 int days = localDow - localizedDayOfWeek(date);  // safe from overflow
985                 date = date.plus(Math.addExact(Math.multiplyExact(weeks, 7), days), DAYS);
986             } else {
987                 int womInt = range.checkValidIntValue(woy, this);  // validate
988                 int weeks = (int) (womInt - localizedWeekOfYear(date));  // safe from overflow
989                 int days = localDow - localizedDayOfWeek(date);  // safe from overflow
990                 date = date.plus(weeks * 7 + days, DAYS);
991                 if (resolverStyle == ResolverStyle.STRICT && date.getLong(YEAR) != year) {
992                     throw new DateTimeException("Strict mode rejected resolved date as it is in a different year");
993                 }
994             }
995             fieldValues.remove(this);
996             fieldValues.remove(YEAR);
997             fieldValues.remove(DAY_OF_WEEK);
998             return date;
999         }

1001         private ChronoLocalDate resolveWBY(
1002                 Map<TemporalField, Long> fieldValues, Chronology chrono, int localDow, ResolverStyle resolverStyle) {
1003             int yowby = weekDef.weekBasedYear.range().checkValidIntValue(
1004                     fieldValues.get(weekDef.weekBasedYear), weekDef.weekBasedYear);
1005             ChronoLocalDate date;
1006             if (resolverStyle == ResolverStyle.LENIENT) {
1007                 date = ofWeekBasedYear(chrono, yowby, 1, localDow);
1008                 long wowby = fieldValues.get(weekDef.weekOfWeekBasedYear);
1009                 long weeks = Math.subtractExact(wowby, 1);
1010                 date = date.plus(weeks, WEEKS);
1011             } else {
1012                 int wowby = weekDef.weekOfWeekBasedYear.range().checkValidIntValue(
1013                         fieldValues.get(weekDef.weekOfWeekBasedYear), weekDef.weekOfWeekBasedYear);  // validate
1014                 date = ofWeekBasedYear(chrono, yowby, wowby, localDow);
1015                 if (resolverStyle == ResolverStyle.STRICT && localizedWeekBasedYear(date) != yowby) {
1016                     throw new DateTimeException("Strict mode rejected resolved date as it is in a different week-based-year");
1017                 }
1018             }
1019             fieldValues.remove(this);
1020             fieldValues.remove(weekDef.weekBasedYear);
1021             fieldValues.remove(weekDef.weekOfWeekBasedYear);
1022             fieldValues.remove(DAY_OF_WEEK);
1023             return date;
1024         }

1087             } else if (rangeUnit == MONTHS) {  // week-of-month

1094                 return YEAR.range();

1096                 throw new IllegalStateException("unreachable, rangeUnit: " + rangeUnit + ", this: " + this);
1097             }
1098         }

1100         /**
1101          * Map the field range to a week range
1102          * @param temporal the temporal
1103          * @param field the field to get the range of
1104          * @return the ValueRange with the range adjusted to weeks.
1105          */
1106         private ValueRange rangeByWeek(TemporalAccessor temporal, TemporalField field) {
1107             int dow = localizedDayOfWeek(temporal);
1108             int offset = startOfWeekOffset(temporal.get(field), dow);
1109             ValueRange fieldRange = temporal.range(field);
1110             return ValueRange.of(computeWeek(offset, (int) fieldRange.getMinimum()),
1111                     computeWeek(offset, (int) fieldRange.getMaximum()));
1112         }

1114         /**
1115          * Map the field range to a week range of a week year.
1116          * @param temporal  the temporal
1117          * @return the ValueRange with the range adjusted to weeks.
1118          */
1119         private ValueRange rangeWeekOfWeekBasedYear(TemporalAccessor temporal) {
1120             if (!temporal.isSupported(DAY_OF_YEAR)) {
1121                 return WEEK_OF_YEAR_RANGE;
1122             }
1123             int dow = localizedDayOfWeek(temporal);
1124             int doy = temporal.get(DAY_OF_YEAR);
1125             int offset = startOfWeekOffset(doy, dow);
1126             int week = computeWeek(offset, doy);
1127             if (week == 0) {
1128                 // Day is in end of week of previous year
1129                 // Recompute from the last day of the previous year
1130                 ChronoLocalDate date = Chronology.from(temporal).date(temporal);
1131                 date = date.minus(doy + 7, DAYS);   // Back down into previous year
1132                 return rangeWeekOfWeekBasedYear(date);
1133             }
1134             // Check if day of year is in partial week associated with next year
1135             ValueRange dayRange = temporal.range(DAY_OF_YEAR);
1136             int yearLen = (int)dayRange.getMaximum();
1137             int newYearWeek = computeWeek(offset, yearLen + weekDef.getMinimalDaysInFirstWeek());
1138 
1139             if (week >= newYearWeek) {
1140                 // Overlaps with weeks of following year; recompute from a week in following year
1141                 ChronoLocalDate date = Chronology.from(temporal).date(temporal);
1142                 date = date.plus(yearLen - doy + 1 + 7, ChronoUnit.DAYS);
1143                 return rangeWeekOfWeekBasedYear(date);
1144             }
1145             return ValueRange.of(1, newYearWeek-1);
1146         }
