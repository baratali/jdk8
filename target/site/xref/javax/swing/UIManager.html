66  /**
67   * {@code UIManager} manages the current look and feel, the set of
68   * available look and feels, {@code PropertyChangeListeners} that
69   * are notified when the look and feel changes, look and feel defaults, and
70   * convenience methods for obtaining various default values.
71   *
72   * <h3>Specifying the look and feel</h3>
73   *
74   * The look and feel can be specified in two distinct ways: by
75   * specifying the fully qualified name of the class for the look and
76   * feel, or by creating an instance of {@code LookAndFeel} and passing
77   * it to {@code setLookAndFeel}. The following example illustrates
78   * setting the look and feel to the system look and feel:
79   * <pre>
80   *   UIManager.setLookAndFeel(UIManager.getSystemLookAndFeelClassName());
81   * </pre>
82   * The following example illustrates setting the look and feel based on
83   * class name:
84   * <pre>
85   *   UIManager.setLookAndFeel("javax.swing.plaf.metal.MetalLookAndFeel");
86   * </pre>
87   * Once the look and feel has been changed it is imperative to invoke
88   * {@code updateUI} on all {@code JComponents}. The method {@link
89   * SwingUtilities#updateComponentTreeUI} makes it easy to apply {@code
90   * updateUI} to a containment hierarchy. Refer to it for
91   * details. The exact behavior of not invoking {@code
92   * updateUI} after changing the look and feel is
93   * unspecified. It is very possible to receive unexpected exceptions,
94   * painting problems, or worse.
95   *
96   * <h3>Default look and feel</h3>
97   *
98   * The class used for the default look and feel is chosen in the following
99   * manner:
100  * <ol>
101  *   <li>If the system property <code>swing.defaultlaf</code> is
102  *       {@code non-null}, use its value as the default look and feel class
103  *       name.
104  *   <li>If the {@link java.util.Properties} file <code>swing.properties</code>
105  *       exists and contains the key <code>swing.defaultlaf</code>,
106  *       use its value as the default look and feel class name. The location
107  *       that is checked for <code>swing.properties</code> may vary depending
108  *       upon the implementation of the Java platform. Typically the
109  *       <code>swing.properties</code> file is located in the <code>lib</code>
110  *       subdirectory of the Java installation directory.
111  *       Refer to the release notes of the implementation being used for
112  *       further details.
113  *   <li>Otherwise use the cross platform look and feel.
114  * </ol>
115  *
116  * <h3>Defaults</h3>
117  *
118  * {@code UIManager} manages three sets of {@code UIDefaults}. In order, they
119  * are:
120  * <ol>
121  *   <li>Developer defaults. With few exceptions Swing does not
122  *       alter the developer defaults; these are intended to be modified
123  *       and used by the developer.
124  *   <li>Look and feel defaults. The look and feel defaults are
125  *       supplied by the look and feel at the time it is installed as the
126  *       current look and feel ({@code setLookAndFeel()} is invoked). The
127  *       look and feel defaults can be obtained using the {@code
128  *       getLookAndFeelDefaults()} method.
129  *   <li>System defaults. The system defaults are provided by Swing.
130  * </ol>
131  * Invoking any of the various {@code get} methods
132  * results in checking each of the defaults, in order, returning
133  * the first {@code non-null} value. For example, invoking
134  * {@code UIManager.getString("Table.foreground")} results in first
135  * checking developer defaults. If the developer defaults contain
136  * a value for {@code "Table.foreground"} it is returned, otherwise
137  * the look and feel defaults are checked, followed by the system defaults.
138  * <p>
139  * It's important to note that {@code getDefaults} returns a custom
140  * instance of {@code UIDefaults} with this resolution logic built into it.
141  * For example, {@code UIManager.getDefaults().getString("Table.foreground")}
142  * is equivalent to {@code UIManager.getString("Table.foreground")}. Both
143  * resolve using the algorithm just described. In many places the
144  * documentation uses the word defaults to refer to the custom instance
145  * of {@code UIDefaults} with the resolution logic as previously described.
146  * <p>
147  * When the look and feel is changed, {@code UIManager} alters only the
148  * look and feel defaults; the developer and system defaults are not
149  * altered by the {@code UIManager} in any way.
150  * <p>
151  * The set of defaults a particular look and feel supports is defined
152  * and documented by that look and feel. In addition, each look and
153  * feel, or {@code ComponentUI} provided by a look and feel, may
154  * access the defaults at different times in their life cycle. Some
155  * look and feels may aggressively look up defaults, so that changing a
156  * default may not have an effect after installing the look and feel.
157  * Other look and feels may lazily access defaults so that a change to
158  * the defaults may effect an existing look and feel. Finally, other look
159  * and feels might not configure themselves from the defaults table in
160  * any way. None-the-less it is usually the case that a look and feel
161  * expects certain defaults, so that in general
162  * a {@code ComponentUI} provided by one look and feel will not
163  * work with another look and feel.
164  * <p>
165  * <strong>Warning:</strong>
166  * Serialized objects of this class will not be compatible with
167  * future Swing releases. The current serialization support is
168  * appropriate for short term storage or RMI between applications running
169  * the same version of Swing.  As of 1.4, support for long term storage
170  * of all JavaBeans&trade;
171  * has been added to the <code>java.beans</code> package.
172  * Please see {@link java.beans.XMLEncoder}.
173  *
174  * @author Thomas Ball
175  * @author Hans Muller
176  */
177 public class UIManager implements Serializable

232     private static final Object classLock = new Object();

234     /**
235      * Return the <code>LAFState</code> object, lazily create one if necessary.
236      * All access to the <code>LAFState</code> fields is done via this method,
237      * for example:
238      * <pre>
239      *     getLAFState().initialized = true;
240      * </pre>
241      */
242     private static LAFState getLAFState() {
243         LAFState rv = (LAFState)SwingUtilities.appContextGet(
244                 SwingUtilities2.LAF_STATE_KEY);
245         if (rv == null) {
246             synchronized (classLock) {
247                 rv = (LAFState)SwingUtilities.appContextGet(
248                         SwingUtilities2.LAF_STATE_KEY);
249                 if (rv == null) {
250                     SwingUtilities.appContextPut(
251                             SwingUtilities2.LAF_STATE_KEY,
252                             (rv = new LAFState()));
253                 }
254             }
255         }
256         return rv;
257     }

280      * The location of the <code>swing.properties</code> property file is

398     /**
399      * Returns an array of {@code LookAndFeelInfo}s representing the
400      * {@code LookAndFeel} implementations currently available. The
401      * <code>LookAndFeelInfo</code> objects can be used by an
402      * application to construct a menu of look and feel options for
403      * the user, or to determine which look and feel to set at startup
404      * time. To avoid the penalty of creating numerous {@code
405      * LookAndFeel} objects, {@code LookAndFeelInfo} maintains the
406      * class name of the {@code LookAndFeel} class, not the actual
407      * {@code LookAndFeel} instance.
408      * <p>
409      * The following example illustrates setting the current look and feel
410      * from an instance of {@code LookAndFeelInfo}:
411      * <pre>
412      *   UIManager.setLookAndFeel(info.getClassName());
413      * </pre>
414      *
415      * @return an array of <code>LookAndFeelInfo</code> objects
416      * @see #setLookAndFeel
417      */
418     public static LookAndFeelInfo[] getInstalledLookAndFeels() {
419         maybeInitialize();
420         LookAndFeelInfo[] ilafs = getLAFState().installedLAFs;
421         if (ilafs == null) {
422             ilafs = installedLAFs;
423         }
424         LookAndFeelInfo[] rv = new LookAndFeelInfo[ilafs.length];
425         System.arraycopy(ilafs, 0, rv, 0, ilafs.length);
426         return rv;
427     }

497     /**
498      * Sets the current look and feel to {@code newLookAndFeel}.
499      * If the current look and feel is {@code non-null} {@code
500      * uninitialize} is invoked on it. If {@code newLookAndFeel} is
501      * {@code non-null}, {@code initialize} is invoked on it followed
502      * by {@code getDefaults}. The defaults returned from {@code
503      * newLookAndFeel.getDefaults()} replace those of the defaults
504      * from the previous look and feel. If the {@code newLookAndFeel} is
505      * {@code null}, the look and feel defaults are set to {@code null}.
506      * <p>
507      * A value of {@code null} can be used to set the look and feel
508      * to {@code null}. As the {@code LookAndFeel} is required for
509      * most of Swing to function, setting the {@code LookAndFeel} to
510      * {@code null} is strongly discouraged.
511      * <p>
512      * This is a JavaBeans bound property.
513      *
514      * @param newLookAndFeel {@code LookAndFeel} to install
515      * @throws UnsupportedLookAndFeelException if
516      *          {@code newLookAndFeel} is {@code non-null} and
517      *          {@code newLookAndFeel.isSupportedLookAndFeel()} returns
518      *          {@code false}
519      * @see #getLookAndFeel
520      */
521     public static void setLookAndFeel(LookAndFeel newLookAndFeel)
522         throws UnsupportedLookAndFeelException
523     {
524         if ((newLookAndFeel != null) && !newLookAndFeel.isSupportedLookAndFeel()) {
525             String s = newLookAndFeel.toString() + " not supported on this platform";
526             throw new UnsupportedLookAndFeelException(s);
527         }
528 
529         LAFState lafState = getLAFState();
530         LookAndFeel oldLookAndFeel = lafState.lookAndFeel;
531         if (oldLookAndFeel != null) {
532             oldLookAndFeel.uninitialize();
533         }
534 
535         lafState.lookAndFeel = newLookAndFeel;
536         if (newLookAndFeel != null) {
537             sun.swing.DefaultLookup.setDefaultLookup(null);
538             newLookAndFeel.initialize();
539             lafState.setLookAndFeelDefaults(newLookAndFeel.getDefaults());
540         }
541         else {
542             lafState.setLookAndFeelDefaults(null);
543         }
544 
545         SwingPropertyChangeSupport changeSupport = lafState.
546                                          getPropertyChangeSupport(false);
547         if (changeSupport != null) {
548             changeSupport.firePropertyChange("lookAndFeel", oldLookAndFeel,
549                                              newLookAndFeel);
550         }
551     }

554     /**
555      * Loads the {@code LookAndFeel} specified by the given class
556      * name, using the current thread's context class loader, and
557      * passes it to {@code setLookAndFeel(LookAndFeel)}.
558      *
559      * @param className  a string specifying the name of the class that implements
560      *        the look and feel
561      * @exception ClassNotFoundException if the <code>LookAndFeel</code>
562      *           class could not be found
563      * @exception InstantiationException if a new instance of the class
564      *          couldn't be created
565      * @exception IllegalAccessException if the class or initializer isn't accessible
566      * @exception UnsupportedLookAndFeelException if
567      *          <code>lnf.isSupportedLookAndFeel()</code> is false
568      * @throws ClassCastException if {@code className} does not identify
569      *         a class that extends {@code LookAndFeel}
570      */
571     public static void setLookAndFeel(String className)
572         throws ClassNotFoundException,
573                InstantiationException,
574                IllegalAccessException,
575                UnsupportedLookAndFeelException
576     {
577         if ("javax.swing.plaf.metal.MetalLookAndFeel".equals(className)) {
578             // Avoid reflection for the common case of metal.
579             setLookAndFeel(new javax.swing.plaf.metal.MetalLookAndFeel());
580         }
581         else {
582             Class lnfClass = SwingUtilities.loadSystemClass(className);
583             setLookAndFeel((LookAndFeel)(lnfClass.newInstance()));
584         }
585     }

587     /**
588      * Returns the name of the <code>LookAndFeel</code> class that implements
589      * the native system look and feel if there is one, otherwise
590      * the name of the default cross platform <code>LookAndFeel</code>
591      * class. This value can be overriden by setting the
592      * <code>swing.systemlaf</code> system property.
593      *
594      * @return the <code>String</code> of the <code>LookAndFeel</code>
595      *          class
596      *
597      * @see #setLookAndFeel
598      * @see #getCrossPlatformLookAndFeelClassName
599      */
600     public static String getSystemLookAndFeelClassName() {
601         String systemLAF = AccessController.doPrivileged(
602                              new GetPropertyAction("swing.systemlaf"));
603         if (systemLAF != null) {
604             return systemLAF;
605         }
606         OSInfo.OSType osType = AccessController.doPrivileged(OSInfo.getOSTypeAction());
607         if (osType == OSInfo.OSType.WINDOWS) {
608             return "com.sun.java.swing.plaf.windows.WindowsLookAndFeel";
609         } else {
610             String desktop = AccessController.doPrivileged(new GetPropertyAction("sun.desktop"));
611             Toolkit toolkit = Toolkit.getDefaultToolkit();
612             if ("gnome".equals(desktop) &&
613                     toolkit instanceof SunToolkit &&
614                     ((SunToolkit) toolkit).isNativeGTKAvailable()) {
615                 // May be set on Linux and Solaris boxs.
616                 return "com.sun.java.swing.plaf.gtk.GTKLookAndFeel";
617             }
618             if (osType == OSInfo.OSType.MACOSX) {

1061      * The auxiliary look and feels tell the multiplexing look and feel what
1062      * other <code>LookAndFeel</code> classes for a component instance are to be used
1063      * in addition to the default <code>LookAndFeel</code> class when creating a
1064      * multiplexing UI.  The change will only take effect when a new
1065      * UI class is created or when the default look and feel is changed
1066      * on a component instance.
1067      * <p>Note these are not the same as the installed look and feels.
1068      *

1073      * @see #getInstalledLookAndFeels
1074      */
1075     static public void addAuxiliaryLookAndFeel(LookAndFeel laf) {
1076         maybeInitialize();
1077 
1078         if (!laf.isSupportedLookAndFeel()) {
1079             // Ideally we would throw an exception here, but it's too late
1080             // for that.
1081             return;
1082         }
1083         Vector<LookAndFeel> v = getLAFState().auxLookAndFeels;
1084         if (v == null) {
1085             v = new Vector<LookAndFeel>();
1086         }
1087 
1088         if (!v.contains(laf)) {
1089             v.addElement(laf);
1090             laf.initialize();
1091             getLAFState().auxLookAndFeels = v;
1092 
1093             if (getLAFState().multiLookAndFeel == null) {
1094                 getLAFState().multiLookAndFeel = getMultiLookAndFeel();
1095             }
1096         }

1102      * other <code>LookAndFeel</code> classes for a component instance are to be used
1103      * in addition to the default <code>LookAndFeel</code> class when creating a
1104      * multiplexing UI.  The change will only take effect when a new
1105      * UI class is created or when the default look and feel is changed
1106      * on a component instance.
1107      * <p>Note these are not the same as the installed look and feels.
1108      * @return true if the <code>LookAndFeel</code> was removed from the list

1112      * @see #getInstalledLookAndFeels

1114     static public boolean removeAuxiliaryLookAndFeel(LookAndFeel laf) {
1115         maybeInitialize();
1116 
1117         boolean result;
1118 
1119         Vector<LookAndFeel> v = getLAFState().auxLookAndFeels;
1120         if ((v == null) || (v.size() == 0)) {
1121             return false;
1122         }
1123 
1124         result = v.removeElement(laf);
1125         if (result) {
1126             if (v.size() == 0) {

1217     private static Properties loadSwingProperties()
1218     {
1219         /* Don't bother checking for Swing properties if untrusted, as
1220          * there's no way to look them up without triggering SecurityExceptions.
1221          */
1222         if (UIManager.class.getClassLoader() != null) {
1223             return new Properties();
1224         }
1225         else {
1226             final Properties props = new Properties();
1227 
1228             java.security.AccessController.doPrivileged(
1229                 new java.security.PrivilegedAction<Object>() {

1276     /**
1277      * If a <code>swing.properties</code> file exist and it has a
1278      * <code>swing.installedlafs</code> property
1279      * then initialize the <code>installedLAFs</code> field.
1280      *
1281      * @see #getInstalledLookAndFeels
1282      */
1283     private static void initializeInstalledLAFs(Properties swingProps)
1284     {
1285         String ilafsString = swingProps.getProperty(installedLAFsKey);
1286         if (ilafsString == null) {
1287             return;
1288         }
1289 
1290         /* Create a vector that contains the value of the swing.installedlafs
1291          * property.  For example given "swing.installedlafs=motif,windows"
1292          * lafs = {"motif", "windows"}.
1293          */
1294         Vector<String> lafs = new Vector<String>();
1295         StringTokenizer st = new StringTokenizer(ilafsString, ",", false);
1296         while (st.hasMoreTokens()) {
1297             lafs.addElement(st.nextToken());
1298         }
1299 
1300         /* Look up the name and class for each name in the "swing.installedlafs"
1301          * list.  If they both exist then add a LookAndFeelInfo to
1302          * the installedLafs array.
1303          */
1304         Vector<LookAndFeelInfo> ilafs = new Vector<LookAndFeelInfo>(lafs.size());
1305         for (String laf : lafs) {
1306             String name = swingProps.getProperty(makeInstalledLAFKey(laf, "name"), laf);
1307             String cls = swingProps.getProperty(makeInstalledLAFKey(laf, "class"));
1308             if (cls != null) {
1309                 ilafs.addElement(new LookAndFeelInfo(name, cls));
1310             }
1311         }
1312 
1313         LookAndFeelInfo[] installedLAFs = new LookAndFeelInfo[ilafs.size()];
1314         for(int i = 0; i < ilafs.size(); i++) {
1315             installedLAFs[i] = ilafs.elementAt(i);
1316         }
1317         getLAFState().installedLAFs = installedLAFs;
1318     }

1321     /**
1322      * If the user has specified a default look and feel, use that.
1323      * Otherwise use the look and feel that's native to this platform.
1324      * If this code is called after the application has explicitly
1325      * set it's look and feel, do nothing.
1326      *
1327      * @see #maybeInitialize
1328      */
1329     private static void initializeDefaultLAF(Properties swingProps)
1330     {
1331         if (getLAFState().lookAndFeel != null) {
1332             return;
1333         }
1334 
1335         // Try to get default LAF from system property, then from AppContext
1336         // (6653395), then use cross-platform one by default.
1337         String lafName = null;
1338         HashMap lafData =
1339                 (HashMap) AppContext.getAppContext().remove("swing.lafdata");
1340         if (lafData != null) {
1341             lafName = (String) lafData.remove("defaultlaf");
1342         }
1343         if (lafName == null) {
1344             lafName = getCrossPlatformLookAndFeelClassName();
1345         }
1346         lafName = swingProps.getProperty(defaultLAFKey, lafName);
1347 
1348         try {
1349             setLookAndFeel(lafName);
1350         } catch (Exception e) {
1351             throw new Error("Cannot load " + lafName);
1352         }
1353 
1354         // Set any properties passed through AppContext (6653395).
1355         if (lafData != null) {
1356             for (Object key: lafData.keySet()) {
1357                 UIManager.put(key, lafData.get(key));
1358             }
1359         }
1360     }

1363     private static void initializeAuxiliaryLAFs(Properties swingProps)
1364     {
1365         String auxLookAndFeelNames = swingProps.getProperty(auxiliaryLAFsKey);
1366         if (auxLookAndFeelNames == null) {
1367             return;
1368         }
1369 
1370         Vector<LookAndFeel> auxLookAndFeels = new Vector<LookAndFeel>();
1371 
1372         StringTokenizer p = new StringTokenizer(auxLookAndFeelNames,",");
1373         String factoryName;
1374 
1375         /* Try to load each LookAndFeel subclass in the list.
1376          */
1377 
1378         while (p.hasMoreTokens()) {
1379             String className = p.nextToken();
1380             try {
1381                 Class lnfClass = SwingUtilities.loadSystemClass(className);
1382                 LookAndFeel newLAF = (LookAndFeel)lnfClass.newInstance();
1383                 newLAF.initialize();
1384                 auxLookAndFeels.addElement(newLAF);
1385             }
1386             catch (Exception e) {
1387                 System.err.println("UIManager: failed loading auxiliary look and feel " + className);
1388             }
1389         }
1390 
1391         /* If there were problems and no auxiliary look and feels were
1392          * loaded, make sure we reset auxLookAndFeels to null.
1393          * Otherwise, we are going to use the MultiLookAndFeel to get
1394          * all component UI's, so we need to load it now.
1395          */
1396         if (auxLookAndFeels.size() == 0) {
1397             auxLookAndFeels = null;
1398         }
1399         else {
1400             getLAFState().multiLookAndFeel = getMultiLookAndFeel();
1401             if (getLAFState().multiLookAndFeel == null) {
1402                 auxLookAndFeels = null;
1403             }
1404         }
1405 
1406         getLAFState().auxLookAndFeels = auxLookAndFeels;
1407     }

1416      * This method is called before any code that depends on the
1417      * <code>AppContext</code> specific LAFState object runs.  When the AppContext
1418      * corresponds to a set of applets it's possible for this method

1453     /*
1454      * Only called by maybeInitialize().
1455      */
1456     private static void initialize() {
1457         Properties swingProps = loadSwingProperties();
1458         initializeSystemDefaults(swingProps);
1459         initializeDefaultLAF(swingProps);
1460         initializeAuxiliaryLAFs(swingProps);
1461         initializeInstalledLAFs(swingProps);
1462 
1463         // Install Swing's PaintEventDispatcher
1464         if (RepaintManager.HANDLE_TOP_LEVEL_PAINT) {
1465             sun.awt.PaintEventDispatcher.setPaintEventDispatcher(
1466                                         new SwingPaintEventDispatcher());
1467         }
1468         // Install a hook that will be invoked if no one consumes the
1469         // KeyEvent.  If the source isn't a JComponent this will process
1470         // key bindings, if the source is a JComponent it implies that
1471         // processKeyEvent was already invoked and thus no need to process
1472         // the bindings again, unless the Component is disabled, in which
1473         // case KeyEvents will no longer be dispatched to it so that we
1474         // handle it here.
1475         KeyboardFocusManager.getCurrentKeyboardFocusManager().
1476                 addKeyEventPostProcessor(new KeyEventPostProcessor() {
1477                     public boolean postProcessKeyEvent(KeyEvent e) {
1478                         Component c = e.getComponent();
1479 
1480                         if ((!(c instanceof JComponent) ||
1481                              (c != null && !c.isEnabled())) &&
1482                                 JComponent.KeyboardState.shouldProcess(e) &&
1483                                 SwingUtilities.processKeyBindings(e)) {
1484                             e.consume();
1485                             return true;
1486                         }
1487                         return false;
1488                     }
1489                 });
1490         AWTAccessor.getComponentAccessor().
1491             setRequestFocusController(JComponent.focusController);
1492     }
