46  /**
47   * The Service Provider Interface (SPI) mechanism that generates <code>SyncProvider</code>
48   * instances to be used by disconnected <code>RowSet</code> objects.
49   * The <code>SyncProvider</code> instances in turn provide the
50   * <code>javax.sql.RowSetReader</code> object the <code>RowSet</code> object
51   * needs to populate itself with data and the
52   * <code>javax.sql.RowSetWriter</code> object it needs to
53   * propagate changes to its
54   * data back to the underlying data source.
55   * <P>
56   * Because the methods in the <code>SyncFactory</code> class are all static,
57   * there is only one <code>SyncFactory</code> object
58   * per Java VM at any one time. This ensures that there is a single source from which a
59   * <code>RowSet</code> implementation can obtain its <code>SyncProvider</code>
60   * implementation.
61   *
62   * <h3>1.0 Overview</h3>
63   * The <code>SyncFactory</code> class provides an internal registry of available
64   * synchronization provider implementations (<code>SyncProvider</code> objects).
65   * This registry may be queried to determine which
66   * synchronization providers are available.
67   * The following line of code gets an enumeration of the providers currently registered.
68   * <PRE>
69   *     java.util.Enumeration e = SyncFactory.getRegisteredProviders();
70   * </PRE>
71   * All standard <code>RowSet</code> implementations must provide at least two providers:
72   * <UL>
73   *  <LI>an optimistic provider for use with a <code>CachedRowSet</code> implementation
74   *     or an implementation derived from it
75   *  <LI>an XML provider, which is used for reading and writing XML, such as with
76   *       <code>WebRowSet</code> objects
77   * </UL>
78   * Note that the JDBC RowSet Implementations include the <code>SyncProvider</code>
79   * implementations <code>RIOptimisticProvider</code> and <code>RIXmlProvider</code>,
80   * which satisfy this requirement.
81   * <P>
82   * The <code>SyncFactory</code> class provides accessor methods to assist
83   * applications in determining which synchronization providers are currently
84   * registered with the <code>SyncFactory</code>.
85   * <p>
86   * Other methods let <code>RowSet</code> persistence providers be
87   * registered or de-registered with the factory mechanism. This
88   * allows additional synchronization provider implementations to be made
89   * available to <code>RowSet</code> objects at run time.
90   * <p>
91   * Applications can apply a degree of filtering to determine the level of
92   * synchronization that a <code>SyncProvider</code> implementation offers.
93   * The following criteria determine whether a provider is
94   * made available to a <code>RowSet</code> object:
95   * <ol>
96   * <li>If a particular provider is specified by a <code>RowSet</code> object, and
97   * the <code>SyncFactory</code> does not contain a reference to this provider,
98   * a <code>SyncFactoryException</code> is thrown stating that the synchronization
99   * provider could not be found.
100  *
101  * <li>If a <code>RowSet</code> implementation is instantiated with a specified
102  * provider and the specified provider has been properly registered, the
103  * requested provider is supplied. Otherwise a <code>SyncFactoryException</code>
104  * is thrown.
105  *
106  * <li>If a <code>RowSet</code> object does not specify a
107  * <code>SyncProvider</code> implementation and no additional
108  * <code>SyncProvider</code> implementations are available, the reference
109  * implementation providers are supplied.
110  * </ol>
111  * <h3>2.0 Registering <code>SyncProvider</code> Implementations</h3>
112  * <p>
113  * Both vendors and developers can register <code>SyncProvider</code>
114  * implementations using one of the following mechanisms.
115  * <ul>
116  * <LI><B>Using the command line</B><BR>
117  * The name of the provider is supplied on the command line, which will add
118  * the provider to the system properties.
119  * For example:
120  * <PRE>
121  *    -Drowset.provider.classname=com.fred.providers.HighAvailabilityProvider
122  * </PRE>
123  * <li><b>Using the Standard Properties File</b><BR>
124  * The reference implementation is targeted
125  * to ship with J2SE 1.5, which will include an additional resource file
126  * that may be edited by hand. Here is an example of the properties file
127  * included in the reference implementation:
128  * <PRE>
129  *   #Default JDBC RowSet sync providers listing
130  *   #
131  *
132  *   # Optimistic synchronization provider
133  *   rowset.provider.classname.0=com.sun.rowset.providers.RIOptimisticProvider
134  *   rowset.provider.vendor.0=Oracle Corporation
135  *   rowset.provider.version.0=1.0
136  *
137  *   # XML Provider using standard XML schema
138  *   rowset.provider.classname.1=com.sun.rowset.providers.RIXMLProvider
139  *   rowset.provider.vendor.1=Oracle Corporation
140  *   rowset.provider.version.1=1.0
141  * </PRE>
142  * The <code>SyncFactory</code> checks this file and registers the
143  * <code>SyncProvider</code> implementations that it contains. A
144  * developer or vendor can add other implementations to this file.
145  * For example, here is a possible addition:
146  * <PRE>
147  *     rowset.provider.classname.2=com.fred.providers.HighAvailabilityProvider
148  *     rowset.provider.vendor.2=Fred, Inc.
149  *     rowset.provider.version.2=1.0
150  * </PRE>
151  *
152  * <li><b>Using a JNDI Context</b><BR>
153  * Available providers can be registered on a JNDI
154  * context, and the <code>SyncFactory</code> will attempt to load
155  * <code>SyncProvider</code> implementations from that JNDI context.
156  * For example, the following code fragment registers a provider implementation
157  * on a JNDI context.  This is something a deployer would normally do. In this
158  * example, <code>MyProvider</code> is being registered on a CosNaming
159  * namespace, which is the namespace used by J2EE resources.
160  * <PRE>
161  *    import javax.naming.*;
162  *
163  *    Hashtable svrEnv = new  Hashtable();
164  *    srvEnv.put(Context.INITIAL_CONTEXT_FACTORY, "CosNaming");
165  *
166  *    Context ctx = new InitialContext(svrEnv);
167  *    com.fred.providers.MyProvider = new MyProvider();
168  *    ctx.rebind("providers/MyProvider", syncProvider);
169  * </PRE>
170  * </ul>
171  * Next, an application will register the JNDI context with the
172  * <code>SyncFactory</code> instance.  This allows the <code>SyncFactory</code>
173  * to browse within the JNDI context looking for <code>SyncProvider</code>
174  * implementations.
175  * <PRE>
176  *    Hashtable appEnv = new Hashtable();
177  *    appEnv.put(Context.INITIAL_CONTEXT_FACTORY, "CosNaming");
178  *    appEnv.put(Context.PROVIDER_URL, "iiop://hostname/providers");
179  *    Context ctx = new InitialContext(appEnv);
180  *
181  *    SyncFactory.registerJNDIContext(ctx);
182  * </PRE>
183  * If a <code>RowSet</code> object attempts to obtain a <code>MyProvider</code>
184  * object, the <code>SyncFactory</code> will try to locate it. First it searches
185  * for it in the system properties, then it looks in the resource files, and
186  * finally it checks the JNDI context that has been set. The <code>SyncFactory</code>
187  * instance verifies that the requested provider is a valid extension of the
188  * <code>SyncProvider</code> abstract class and then gives it to the
189  * <code>RowSet</code> object. In the following code fragment, a new
190  * <code>CachedRowSet</code> object is created and initialized with
191  * <i>env</i>, which contains the binding to <code>MyProvider</code>.
192  * <PRE>
193  *    Hashtable env = new Hashtable();
194  *    env.put(SyncFactory.ROWSET_SYNC_PROVIDER, "com.fred.providers.MyProvider");
195  *    CachedRowSet crs = new com.sun.rowset.CachedRowSetImpl(env);
196  * </PRE>
197  * Further details on these mechanisms are available in the
198  * <code>javax.sql.rowset.spi</code> package specification.
199  *
200  * @author  Jonathan Bruce
201  * @see javax.sql.rowset.spi.SyncProvider
202  * @see javax.sql.rowset.spi.SyncFactoryException
203  */

262      * are provided in the <code>SyncProvider</code> specification for the

265      * <p>
266      * Synchronization providers bound to a JNDI context can be
267      * registered by binding a SyncProvider instance to a JNDI namespace.

269      * <pre>

279      * available <code>SyncProvider</code> objects bound to the JNDI

284      * @throws SyncFactoryException if an attempt is made to supply an empty

290

292         impl.setClassname(providerID);
293         initMapIfNecessary();
294         implementations.put(providerID, impl);

330     private static synchronized void initMapIfNecessary() throws SyncFactoryException {
331 
332         // Local implementation class names and keys from Properties
333         // file, translate names into Class objects using Class.forName
334         // and store mappings
335         final Properties properties = new Properties();
336 
337         if (implementations == null) {
338             implementations = new Hashtable<>();
339 
340             try {
341 
342                 // check if user is supplying his Synchronisation Provider
343                 // Implementation if not using Oracle's implementation.
344                 // properties.load(new FileInputStream(ROWSET_PROPERTIES));
345 
346                 // The rowset.properties needs to be in jdk/jre/lib when
347                 // integrated with jdk.
348                 // else it should be picked from -D option from command line.
349 
350                 // -Drowset.properties will add to standard properties. Similar
351                 // keys will over-write
352 
353                 /*
354                  * Dependent on application
355                  */
356                 String strRowsetProperties;
357                 try {
358                     strRowsetProperties = AccessController.doPrivileged(new PrivilegedAction<String>() {
359                         public String run() {
360                             return System.getProperty("rowset.properties");
361                         }
362                     }, null, new PropertyPermission("rowset.properties", "read"));
363                 } catch (Exception ex) {
364                     System.out.println("errorget rowset.properties: " + ex);
365                     strRowsetProperties = null;
366                 };
367 
368                 if (strRowsetProperties != null) {
369                     // Load user's implementation of SyncProvider
370                     // here. -Drowset.properties=/abc/def/pqr.txt
371                     ROWSET_PROPERTIES = strRowsetProperties;
372                     try (FileInputStream fis = new FileInputStream(ROWSET_PROPERTIES)) {
373                         properties.load(fis);
374                     }
375                     parseProperties(properties);
376                 }
377 
378                 /*
379                  * Always available
380                  */
381                 ROWSET_PROPERTIES = "javax" + strFileSep + "sql" +
382                         strFileSep + "rowset" + strFileSep +
383                         "rowset.properties";
384 
385                 ClassLoader cl = Thread.currentThread().getContextClassLoader();
386 
387                 try {
388                     AccessController.doPrivileged((PrivilegedExceptionAction<Void>) () -> {
389                         try (InputStream stream = (cl == null) ?
390                                 ClassLoader.getSystemResourceAsStream(ROWSET_PROPERTIES)
391                                 : cl.getResourceAsStream(ROWSET_PROPERTIES)) {
392                             if (stream == null) {
393                                 throw new SyncFactoryException("Resource " + ROWSET_PROPERTIES + " not found");
394                             }
395                             properties.load(stream);
396                         }
397                         return null;
398                     });
399                 } catch (PrivilegedActionException ex) {
400                     Throwable e = ex.getException();
401                     if (e instanceof SyncFactoryException) {
402                       throw (SyncFactoryException) e;
403                     } else {
404                         SyncFactoryException sfe = new SyncFactoryException();
405                         sfe.initCause(ex.getException());
406                         throw sfe;
407                     }
408                 }
409 
410                 parseProperties(properties);
411 
412             // removed else, has properties should sum together
413 
414             } catch (FileNotFoundException e) {
415                 throw new SyncFactoryException("Cannot locate properties file: " + e);
416             } catch (IOException e) {
417                 throw new SyncFactoryException("IOException: " + e);
418             }
419 
420             /*
421              * Now deal with -Drowset.provider.classname
422              * load additional properties from -D command line
423              */
424             properties.clear();
425             String providerImpls;
426             try {
427                 providerImpls = AccessController.doPrivileged(new PrivilegedAction<String>() {
428                     public String run() {
429                         return System.getProperty(ROWSET_SYNC_PROVIDER);
430                     }
431                 }, null, new PropertyPermission(ROWSET_SYNC_PROVIDER, "read"));
432             } catch (Exception ex) {
433                 providerImpls = null;
434             }
435 
436             if (providerImpls != null) {
437                 int i = 0;
438                 if (providerImpls.indexOf(colon) > 0) {
439                     StringTokenizer tokenizer = new StringTokenizer(providerImpls, colon);
440                     while (tokenizer.hasMoreElements()) {
441                         properties.put(ROWSET_SYNC_PROVIDER + "." + i, tokenizer.nextToken());
442                         i++;
443                     }
444                 } else {
445                     properties.put(ROWSET_SYNC_PROVIDER, providerImpls);
446                 }
447                 parseProperties(properties);
448             }
449         }
450     }

462     /**
463      * Internal handler for all standard property parsing. Parses standard
464      * ROWSET properties and stores lazy references into the the internal registry.
465      */
466     private static void parseProperties(Properties p) {
467 
468         ProviderImpl impl = null;
469         String key = null;
470         String[] propertyNames = null;
471 
472         for (Enumeration<?> e = p.propertyNames(); e.hasMoreElements();) {
473 
474             String str = (String) e.nextElement();
475 
476             int w = str.length();
477 
478             if (str.startsWith(SyncFactory.ROWSET_SYNC_PROVIDER)) {
479 
480                 impl = new ProviderImpl();
481                 impl.setIndex(providerImplIndex++);
482 
483                 if (w == (SyncFactory.ROWSET_SYNC_PROVIDER).length()) {
484                     // no property index has been set.
485                     propertyNames = getPropertyNames(false);
486                 } else {
487                     // property index has been set.
488                     propertyNames = getPropertyNames(true, str.substring(w - 1));
489                 }
490 
491                 key = p.getProperty(propertyNames[0]);
492                 impl.setClassname(key);
493                 impl.setVendor(p.getProperty(propertyNames[1]));
494                 impl.setVersion(p.getProperty(propertyNames[2]));
495                 implementations.put(key, impl);
496             }
497         }
498     }

500     /**
501      * Used by the parseProperties methods to disassemble each property tuple.

503     private static String[] getPropertyNames(boolean append) {

507     /**
508      * Disassembles each property and its associated value. Also handles
509      * overloaded property names that contain indexes.
510      */
511     private static String[] getPropertyNames(boolean append,
512             String propertyIndex) {
513         String dot = ".";
514         String[] propertyNames =
515                 new String[]{SyncFactory.ROWSET_SYNC_PROVIDER,
516             SyncFactory.ROWSET_SYNC_VENDOR,
517             SyncFactory.ROWSET_SYNC_PROVIDER_VERSION};
518         if (append) {
519             for (int i = 0; i < propertyNames.length; i++) {
520                 propertyNames[i] = propertyNames[i] +
521                         dot +
522                         propertyIndex;
523             }
524             return propertyNames;
525         } else {
526             return propertyNames;
527         }
528     }

541     /**
542      * Returns the <code>SyncProvider</code> instance identified by <i>providerID</i>.
543      *
544      * @param providerID the unique identifier of the provider
545      * @return a <code>SyncProvider</code> implementation
546      * @throws SyncFactoryException If the SyncProvider cannot be found,
547      * the providerID is {@code null}, or
548      * some error was encountered when trying to invoke this provider.
549      */
550     public static SyncProvider getInstance(String providerID)
551             throws SyncFactoryException {
552 
553         if(providerID == null) {
554             throw new SyncFactoryException("The providerID cannot be null");
555         }
556 
557         initMapIfNecessary(); // populate HashTable
558         initJNDIContext();    // check JNDI context for any additional bindings
559 
560         ProviderImpl impl = (ProviderImpl) implementations.get(providerID);
561 
562         if (impl == null) {
563             // Requested SyncProvider is unavailable. Return default provider.
564             return new com.sun.rowset.providers.RIOptimisticProvider();
565         }
566 
567         try {
568             ReflectUtil.checkPackageAccess(providerID);
569         } catch (java.security.AccessControlException e) {
570             SyncFactoryException sfe = new SyncFactoryException();
571             sfe.initCause(e);
572             throw sfe;
573         }
574 
575         // Attempt to invoke classname from registered SyncProvider list
576         Class<?> c = null;
577         try {
578             ClassLoader cl = Thread.currentThread().getContextClassLoader();
579 
580             /**
581              * The SyncProvider implementation of the user will be in
582              * the classpath. We need to find the ClassLoader which loads
583              * this SyncFactory and try to load the SyncProvider class from
584              * there.
585              **/
586             c = Class.forName(providerID, true, cl);
587 
588             if (c != null) {
589                 return (SyncProvider) c.newInstance();
590             } else {
591                 return new com.sun.rowset.providers.RIOptimisticProvider();
592             }
593 
594         } catch (IllegalAccessException e) {
595             throw new SyncFactoryException("IllegalAccessException: " + e.getMessage());
596         } catch (InstantiationException e) {
597             throw new SyncFactoryException("InstantiationException: " + e.getMessage());
598         } catch (ClassNotFoundException e) {
599             throw new SyncFactoryException("ClassNotFoundException: " + e.getMessage());
600         }
601     }

626      * Sets the logging object to be used by the <code>SyncProvider</code>
627      * implementation provided by the <code>SyncFactory</code>. All
628      * <code>SyncProvider</code> implementations can log their events to
629      * this object and the application can retrieve a handle to this
630      * object using the <code>getLogger</code> method.
631      * <p>
632      * This method checks to see that there is an {@code SQLPermission}
633      * object  which grants the permission {@code setSyncFactory}
634      * before allowing the method to succeed.  If a
635      * {@code SecurityManager} exists and its
636      * {@code checkPermission} method denies calling {@code setLogger},
637      * this method throws a
638      * {@code java.lang.SecurityException}.
639      *
640      * @param logger A Logger object instance
641      * @throws java.lang.SecurityException if a security manager exists and its
642      *   {@code checkPermission} method denies calling {@code setLogger}
643      * @throws NullPointerException if the logger is null
644      * @see SecurityManager#checkPermission
645      */
646     public static void setLogger(Logger logger) {
647 
648         SecurityManager sec = System.getSecurityManager();
649         if (sec != null) {
650             sec.checkPermission(SET_SYNCFACTORY_PERMISSION);
651         }
652 
653         if(logger == null){
654             throw new NullPointerException("You must provide a Logger");
655         }
656         rsLogger = logger;
657     }

659     /**
660      * Sets the logging object that is used by <code>SyncProvider</code>
661      * implementations provided by the <code>SyncFactory</code> SPI. All
662      * <code>SyncProvider</code> implementations can log their events
663      * to this object and the application can retrieve a handle to this
664      * object using the <code>getLogger</code> method.
665      * <p>
666      * This method checks to see that there is an {@code SQLPermission}
667      * object  which grants the permission {@code setSyncFactory}
668      * before allowing the method to succeed.  If a
669      * {@code SecurityManager} exists and its
670      * {@code checkPermission} method denies calling {@code setLogger},
671      * this method throws a
672      * {@code java.lang.SecurityException}.
673      *
674      * @param logger a Logger object instance
675      * @param level a Level object instance indicating the degree of logging
676      * required
677      * @throws java.lang.SecurityException if a security manager exists and its
678      *   {@code checkPermission} method denies calling {@code setLogger}
679      * @throws NullPointerException if the logger is null
680      * @see SecurityManager#checkPermission
681      * @see LoggingPermission
682      */
683     public static void setLogger(Logger logger, Level level) {
684         // singleton
685         SecurityManager sec = System.getSecurityManager();
686         if (sec != null) {
687             sec.checkPermission(SET_SYNCFACTORY_PERMISSION);
688         }
689 
690         if(logger == null){
691             throw new NullPointerException("You must provide a Logger");
692         }
693         logger.setLevel(level);
694         rsLogger = logger;
695     }

745     /**
746      * Controls JNDI context initialization.
747      *
748      * @throws SyncFactoryException if an error occurs parsing the JNDI context
749      */
750     private static synchronized void initJNDIContext() throws SyncFactoryException {
751 
752         if ((ic != null) && (lazyJNDICtxRefresh == false)) {
753             try {
754                 parseProperties(parseJNDIContext());
755                 lazyJNDICtxRefresh = true; // touch JNDI namespace once.
756             } catch (NamingException e) {
757                 e.printStackTrace();
758                 throw new SyncFactoryException("SPI: NamingException: " + e.getExplanation());
759             } catch (Exception e) {
760                 e.printStackTrace();
761                 throw new SyncFactoryException("SPI: Exception: " + e.getMessage());
762             }
763         }
764     }
765     /**
766      * Internal switch indicating whether the JNDI namespace should be re-read.
767      */
768     private static boolean lazyJNDICtxRefresh = false;

770     /**
771      * Parses the set JNDI Context and passes bindings to the enumerateBindings
772      * method when complete.
773      */
774     private static Properties parseJNDIContext() throws NamingException {
775 
776         NamingEnumeration<?> bindings = ic.listBindings("");
777         Properties properties = new Properties();
778 
779         // Hunt one level below context for available SyncProvider objects
780         enumerateBindings(bindings, properties);
781 
782         return properties;
783     }
784 
785     /**
786      * Scans each binding on JNDI context and determines if any binding is an
787      * instance of SyncProvider, if so, add this to the registry and continue to
788      * scan the current context using a re-entrant call to this method until all
789      * bindings have been enumerated.
790      */
791     private static void enumerateBindings(NamingEnumeration<?> bindings,
792             Properties properties) throws NamingException {
793 
794         boolean syncProviderObj = false; // move to parameters ?
795 
796         try {
797             Binding bd = null;
798             Object elementObj = null;
799             String element = null;
800             while (bindings.hasMore()) {
801                 bd = (Binding) bindings.next();
802                 element = bd.getName();
803                 elementObj = bd.getObject();
804 
805                 if (!(ic.lookup(element) instanceof Context)) {
806                     // skip directories/sub-contexts
807                     if (ic.lookup(element) instanceof SyncProvider) {
808                         syncProviderObj = true;
809                     }
810                 }
811 
812                 if (syncProviderObj) {
813                     SyncProvider sync = (SyncProvider) elementObj;
814                     properties.put(SyncFactory.ROWSET_SYNC_PROVIDER,
815                             sync.getProviderID());
816                     syncProviderObj = false; // reset
817                 }
818 
819             }
820         } catch (javax.naming.NotContextException e) {
821             bindings.next();
822             // Re-entrant call into method
823             enumerateBindings(bindings, properties);
824         }
825     }
